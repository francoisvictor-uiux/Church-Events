### آلية التسجيل

| الدور | يسجل حضور | كيف |
|-------|-----------|-----|
| أمين الخدمة | الخدام | QR scan أو تحديد يدوي |
| الخادم | مخدوميه فقط | QR scan أو تحديد يدوي |
| المخدوم | **لا أحد** | يعرض QR فقط ليمسحه الخادم |

### ما يرى المخدوم بدلاً من "سجل حضوري":

زر **"أظهر بطاقتي"** يفتح QR شخصي كبير على الشاشة ليمسحه الخادم. فور مسح الخادم → إشعار فوري للمخدوم بتأكيد الحضور + النقاط المكتسبة.

## 5. هيكل البيانات — حقول كل كيان

### 5.1 المستخدم (User)

**الحقول الأساسية (إلزامية):**
- الاسم الكامل
- رقم الهاتف (Egyptian +20)
- الدور: خادم / مخدوم
- الخادم المسؤول عنه (إذا مخدوم)
- المجموعة

**الحقول التفصيلية (اختيارية — يكملها المستخدم):**
- الإيميل
- الرقم القومي
- العنوان بالتفصيل (محافظة / مدينة / حي / شارع)
- تاريخ الميلاد
- الدفعة (سنة الانضمام)
- الكلية / العمل
- الحالة الاجتماعية

**الحقول الحسابية (System-generated):**
- كود QR فريد وثابت (لا يتغير)
- نوع المستخدم (خادم / مخدوم)
- الرصيد الحالي — نقاط
- الرصيد الحالي — مادي (جنيهات معادلة)
- مرات الحضور الإجمالية / الغياب
- إجمالي النقاط المكتسبة / المخصومة / المصروفة
- تاريخ آخر ظهور في حدث
- Streak الحالي (أسابيع متتالية)

---

### 5.2 الحدث (Event)

**المعلومات الأساسية:**
- عنوان الحدث
- البرنامج: قائمة مرحلية [(وقت — اسم النشاط — تعليمات), ...]
- الاشتراك (هل مدفوع؟)
- تفاصيل (وصف نصي)
- تاريخ البداية → تاريخ النهاية
- السعر
- طريقة الدفع
- المدعوون: الكل / مجموعات / أفراد محددون
- الرقم السري للدخول (اختياري)
- الكلية / النادي (اختياري)
- رقم السرى

**الحالات المزدوجة:**
- حالة التسجيل: مفتوح / قابل للحجز / محجوز
- حالة العرض للمنتظرين: مفتوح / المحجوز / المنتظر / المسجلين

**المحتوى:**
- صور الحدث (حتى 10 صور)
- ملخصات الحدث (نص)
- ملفات صوتية / بودكاست
- تنبيهات مخصصة للحدث

---

### 5.3 الحضور (Attendance Record)

- معرّف الحدث
- معرّف المستخدم
- حالة: حضر / غاب / معذور
- وقت التسجيل
- سجّله: [معرف الخادم/الأمين الذي سجّل]
- طريقة التسجيل: QR / يدوي
- ملاحظة (اختيارية)

---

### 5.4 المشتركون في الحدث (Event Participants)

- رقم المستخدم
- الاسم
- رقم الهاتف
- الحالة (مسجل / منتظر / ملغي)
- الرصيد الحالي قبل الحدث
- الرصيد المطلوب للحدث
- مرات الحضور في هذا الحدث (حضور / غياب)

---

### 5.5 المهمة (Task)

- الحدث المرتبط (اختياري)
- نوع المُكلَّف: خادم / مخدوم
- المُكلَّف: فرد / مجموعة / الكل
- نوع المهمة: روحية / قراءة أجندة / مسابقة / استبيان / أخرى
- شرح المهمة (نص)
- المطلوب من المخدوم:
  - إجابة نصية
  - اختيارات (Multiple choice)
  - رفع صور أو ملفات
  - إدراج رابط
- من تاريخ → إلى تاريخ / كل...
- دورية: مرة واحدة / أسبوعي / شهري
- عدد النقاط عند الإنجاز
- إضافة النقاط: تلقائي فور الإرسال / يدوي بعد تقييم الخادم
- حالة المهمة: لم تبدأ / قيد التنفيذ / منجزة / مرفوضة

**للاستبيانات:**
- سؤال
- الاختيارات [(نص, قيمة), ...]

---

### 5.6 تنفيذ المهمة (Task Submission)

- المشترك (من نفّذ)
- رقم المهمة
- رقم النموذج / الإرسال
- حالة: منتظر / مقبول / مرفوض
- ملاحظات الخادم
- كود QR مرتبط بالتسليم (للتحقق الفيزيائي)

---

### 5.7 جدول النقاط (Points Ledger)

- نوع النقطة: حضور / تنفيذ مهمة / مكافأة / خصم / صرف
- نوع العملية: إضافة ✚ / خصم ✕ / صرف
- عدد النقاط
- كود QR مرتبط بالحدث (إن وجد)
- المعتمد (من أضاف/خصم)
- سبب العملية
- ملاحظات
- الطابع الزمني

---

### 5.8 صندوق الخدمة (Fund)

**دخل:**
- النوع: تبرع / رسوم حدث / أخرى
- الحدث المرتبط
- المعتمد (المعطي)
- التاريخ
- المبلغ
- ملاحظات

**مصروف:**
- النوع: أجرة / طعام / طباعة / مستلزمات / أخرى
- الحدث المرتبط
- المعتمد (المُصرِّف)
- التاريخ
- المبلغ
- المتبقي (يُحسب تلقائياً)

## 15. متطلبات غير وظيفية

### 15.1 الأداء

| المعيار | القيمة |
|---------|--------|
| تحميل أي شاشة | ≤ 2 ثانية على 4G |
| استجابة أي action | ≤ 3 ثوانٍ |
| مسح QR | ≤ 1 ثانية |
| Offline mode | عرض البيانات المحفوظة — لا كتابة |

### 15.2 الأمان

- OTP على كل تسجيل دخول (بدون كلمات سر)
- Token refresh تلقائي
- بيانات مشفرة في قاعدة البيانات
- الخادم لا يرى مخدومي مجموعات أخرى أبداً
- QR ديناميكي (يتجدد كل 5 دقائق) لمنع الاستخدام المتكرر
- كل event مسجل بـ (من سجّله + الوقت + الطريقة)

### 15.3 التصميم والـ UX

| المعيار | القيمة |
|---------|--------|
| الاتجاه | RTL كامل على مستوى الجذر |
| الخط | Changa (عربي وإنجليزي) |
| Touch target | 44px × 44px حداً أدنى |
| دعم iOS | iOS 14+ |
| دعم Android | Android 8+ |
| Themes | Light Mode + Dark Mode (v2) |

### 15.4 إمكانية الوصول

- WCAG AA — contrast ratio 4.5:1 كحد أدنى
- دعم Dynamic Font Size على iOS
- VoiceOver / TalkBack للعناصر الأساسية
- Haptic feedback على أحداث مهمة (حضور / نقاط)

### 15.5 مبادئ UX الأساسية

1. **السرعة فوق الجمال** — كل action ≤ 3 ثوانٍ
2. **CTA واحد واضح في كل شاشة** — لا تشتيت
3. **تقليل الـ Forms** — الحقول الاختيارية لاحقاً
4. **التحفيز البصري** — كل نقطة مكتسبة = animation
5. **الهرمية واضحة** — المخدوم لا يرى زر تسجيل حضور
